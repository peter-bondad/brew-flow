# Git Workflow

Use Conventional Commits.

Format:

type(scope): summary

Examples:

feat(inventory): add stock adjustment dialog

fix(auth): prevent expired session redirect

refactor(database): simplify inventory repository

perf(api): optimize ingredient query

docs(api): update supplier documentation

Generate one commit message after every completed task.

Use:

- feat
- fix
- refactor
- perf
- docs
- test
- chore
- style
- ci
- build

Keep summaries concise, imperative, and under 72 characters.
